🔄 ReplayBuffer 工作原理可视化

本文件配套 rsl_rl/storage/replay_buffer.py。ReplayBuffer 在 AMP 项目中用于存 判别器训练用的策略 (s, s') 转移对，是个固定大小的环形缓冲区（FIFO）。

📦 1. 整体结构（数据结构）

三个关键变量：
  • states / next_states：两个预分配的 (buffer_size, obs_dim) 大张量
  • step：当前写入位置（指针）
  • num_samples：当前有效数据量（不超过 buffer_size）

✏️ 2. insert(states, next_states) —— 写入（环形）

情况 A：正常写入（不需回绕）

假设 buffer_size=10, num_samples=7, step=7，要插入 2 个新状态 [s7, s8]：

num_states = 2
start_idx = 7
end_idx = 9   # 9 < 10, 不需要回绕, 走 else 分支

代码路径：else 分支（end_idx < buffer_size）：

self.states[7:9] = states
self.next_states[7:9] = next_states
self.step = (7 + 2) % 10 = 9       # step 移动
self.num_samples = max(9, 7) = 9    # 增加

情况 B：跨越边界（要回绕）

假设 step=7, num_samples=7，这次要插入 5 个 新状态 [s7, s8, s9, s10, s11]：

num_states = 5
start_idx = 7
end_idx = 7 + 5 = 12  # 12 > 10, 要回绕!
self.states[7:10] = states[:3] ← 写入 s7, s8, s9 s0 s1 s2 s3 s4 s5 s6 s7 s8 s9 第 2 段：从 0 开始填剩下的（位置 0, 1） 代码：self.states[0:2] = states[3:] ← 写入 s10, s11 ⚠️ 位置 0, 1 原本是 s0, s1，被覆盖了！ s10 覆盖! s11 覆盖! s2 s3 s4 s5 s6 s7 s8 s9 step=2 (回绕后) num_samples = 10 (满) step = (7+5) % 10 = 2 FIFO: s0, s1 已被淘汰 📌 关键观察 • 5 个新数据分两段写：先写 3 个到末尾（位置 7,8,9），再写 2 个到头部（位置 0,1） • end_idx - buffer_size = 12 - 10 = 2 就是回绕后写入的个数 • buffer_size - self.step = 10 - 7 = 3 就是末尾那段的长度
⚠️ FIFO 行为：当 buffer 满后插入新数据会覆盖最老的数据。这里 s0, s1 是最老的，被 s10, s11 覆盖。

代码逻辑总结

num_states = states.shape[0]
start_idx = self.step
end_idx = self.step + num_states

if end_idx > self.buffer_size:        # 情况 B: 跨越边界
    末尾段 = self.states[step : buffer_size]    # 填到 buffer 末尾
    头部段 = self.states[0 : end_idx - buffer_size]  # 从 0 开始
else:                                  # 情况 A: 正常
    self.states[step:end_idx] = states

self.num_samples = min(buffer_size, max(end_idx, num_samples))  # 取封顶值
self.step = (step + num_states) % buffer_size  # 步进, 满了回绕

🎲 3. feed_forward_generator(num_mini_batch, mini_batch_size) —— 采样

这是个生成器（用 yield），每次 next() 返回一个 mini-batch。

[0, num_samples) 随机抽, 不从 [0, buffer_size)

代码逐行解读

def feed_forward_generator(self, num_mini_batch, mini_batch_size):
    """生成 num_mini_batch 个 mini-batch, 每个有 mini_batch_size 个样本."""
    for _ in range(num_mini_batch):
        # 1. 随机抽索引 (有放回, np.random.choice 默认 replace=True)
        sample_idxs = np.random.choice(self.num_samples, size=mini_batch_size)
        #    e.g. 第一次 yield 抽到 [2, 5, 0], 第二次可能 [4, 4, 1] (有重复)

        # 2. 用 fancy indexing 取出对应位置的数据
        #    states[[2,5,0]] 同时取 idx=2, idx=5, idx=0 的数据
        # 3. yield 返回 (states_batch, next_states_batch)
        yield (
            self.states[sample_idxs].to(self.device),
            self.next_states[sample_idxs].to(self.device),
        )

为什么是生成器（yield）？

如果用 return [...]，会一次性生成所有 mini-batch，可能占用巨大内存。

用 yield，每次 next() 才生成一个 batch，用完即丢。

关键设计

设计代码作用
有放回抽样np.random.choice(num_samples, size=N)实现简单, 重复不影响训练
从 num_samples 抽不是 buffer_size不抽到空位置 (未填满时)
同索引双取states[idxs], next_states[idxs]保证 (s, s') 配对正确
yield生成器节省内存, 按需生产

🔁 4. insert 与 feed_forward 的关系

被动：每步自动写入, 不需要主动调用 • feed_forward_generator 是主动：update() 时由训练循环拉取 • 两者解耦：写入和采样可以同时进行（虽然不真并行）

💡 5. 为什么 AMP 需要 ReplayBuffer？

核心矛盾：判别器需要看 G1 的"整体历史"才能判断它像不像人，不能只看最近几步。

只用最近 batch（不用 ReplayBuffer）用 ReplayBuffer
判别器看到的数据最新 T×E 步 (~24k 步)最近 100k 步
判别器能学到的"G1 当前这步像不像""G1 整个分布像不像人"
判别器稳定性容易过拟合当前策略稳定, 看到完整历史
G1 训练效果策略震荡, 不稳定风格奖励稳定, 训练平滑

类比：像教学生写作文。

• 不用 ReplayBuffer：老师只看学生最近一篇作文就评分 → 评分波动大, 学生摸不到规律

• 用 ReplayBuffer：老师看学生最近 1000 篇作文, 找"整体风格" → 评分稳定, 学生有明确改进方向

📝 6. 完整代码

import numpy as np
import torch


class ReplayBuffer:
    """Fixed-size buffer to store experience tuples."""

    def __init__(self, obs_dim, buffer_size, device):
        """初始化一个 ReplayBuffer 对象.

        Args:
            obs_dim (int): 单个状态的维度 (如 195)
            buffer_size (int): buffer 最大容量 (如 100000)
            device (torch.device): 张量所在设备
        """
        # 预分配两个大张量 (避免训练时反复分配)
        self.states      = torch.zeros(buffer_size, obs_dim).to(device)
        self.next_states = torch.zeros(buffer_size, obs_dim).to(device)
        self.buffer_size = buffer_size
        self.device      = device

        self.step        = 0   # 当前写入位置 (环形指针)
        self.num_samples = 0   # 当前有效样本数

    def insert(self, states, next_states):
        """添加新的 (state, next_state) 对到 buffer (FIFO 环形)."""
        num_states = states.shape[0]
        start_idx  = self.step
        end_idx    = self.step + num_states

        if end_idx > self.buffer_size:
            # 情况 B: 跨越边界, 分两段写
            # 末尾段: 从 step 写到 buffer 末尾
            self.states[      self.step : self.buffer_size] = states[      : self.buffer_size - self.step]
            self.next_states[ self.step : self.buffer_size] = next_states[ : self.buffer_size - self.step]
            # 头部段: 从 0 开始写剩下的
            self.states[      : end_idx - self.buffer_size] = states[      self.buffer_size - self.step :]
            self.next_states[ : end_idx - self.buffer_size] = next_states[ self.buffer_size - self.step :]
        else:
            # 情况 A: 正常连续写
            self.states[      start_idx:end_idx] = states
            self.next_states[ start_idx:end_idx] = next_states

        # 更新计数 (num_samples 不会超过 buffer_size)
        self.num_samples = min(self.buffer_size, max(end_idx, self.num_samples))
        # step 移动, 自动回绕
        self.step = (self.step + num_states) % self.buffer_size

    def feed_forward_generator(self, num_mini_batch, mini_batch_size):
        """生成 num_mini_batch 个 mini-batch, 每个有 mini_batch_size 个样本.

        用 yield 逐个生成, 避免一次性占用大内存.
        """
        for _ in range(num_mini_batch):
            # 从 [0, num_samples) 范围随机抽样 (有放回)
            sample_idxs = np.random.choice(self.num_samples, size=mini_batch_size)
            # 用 fancy indexing 取出对应位置的 (s, s') 对
            yield (
                self.states[sample_idxs].to(self.device),
                self.next_states[sample_idxs].to(self.device),
            )

🎯 7. 一句话总结

ReplayBuffer 是个 FIFO 环形缓冲区，预分配两个 (buffer_size, obs_dim) 张量存 (s, s') 对；
insert 按 step 指针写入，满了就回绕覆盖最老数据；
feed_forward_generator 用 yield 逐个随机采样 mini-batch 给判别器训练用。

核心目的：让判别器看到 G1 的"完整历史分布"（100k 步），而不是只看最近 batch，从而学到稳定的"什么像人"标准。
