问题 1: D 太强为何崩 问题 2: 不用更好奖励? 两题关系 总结

🎯 两个核心问题解答

D 太强的根因 & 为什么不让奖励自带梯度

问题 1: D 太强的表现 —— 奖励几乎为 0

1.1 表面原因：clip(min=0) 把奖励压到 0

风格奖励公式：

style_reward = clip(1 - 0.25 · (d - 1)², min=0)

当 D 太强时，d 接近 -1（比如 d = -0.95）：

1 - 0.25 · (-0.95 - 1)²
= 1 - 0.25 · 3.8025
= 1 - 0.95
= 0.05    ← 这个值大于 0, 没被 clip 掉!

等一下，clip(min=0) 不是根因。即使没有 clip，0.05 本身也已经很小了。

1.2 真正原因：平方函数天然惩罚远离 +1

把奖励函数画出来：

这是一条凸函数，离 d = +1 越远，奖励越低。设 \(x = d - 1\)：

  • \(x = 0\)（d = 1）：reward = 1（满分）
  • \(x = -1\)（d = 0）：reward = 0.75
  • \(x = -2\)（d = -1）：reward = 0

关键问题：当 d = -0.95 时，奖励只有 0.05——G1 收到的信号极弱。

1.3 D 太强时的真正问题：奖励绝对值小 + G1 难逃低分区

对 reward 关于 d 求导：

$$\frac{d}{dd} r_{\text{style}} = -0.5(d - 1) = 0.5(1 - d)$$

关键纠正：局部斜率在 d=-1 处是 1.0（最大），不是 0。前面说"梯度消失"是误导。真实问题分两层：

问题 1：绝对值小

G1 在 d=-0.95 时拿到的 reward = 0.05——无论怎么局部斜率，绝对值几乎为 0。PPO 看的是绝对值，不是斜率。

问题 2：G1 难逃低分区

D 太强时，G1 长期处于 d ∈ [-1, -0.5]——这是 D 判定的"明显不像人"区。要让 d 跳出这个区需要极大努力，G1 努力 0.1 只能爬到 -0.85，永远拿不到 0.8+ 的高分。

+0.80

拖动改变 d，看 reward 绝对值和"对改进 0.1 的敏感度"

1 d = +0.8（健康）：reward = 0.98，改进 0.1 后 0.99（提升 0.01，但起点已很高）

2 d = 0（中等）：reward = 0.75，改进 0.1 后 0.80（提升 0.05）

3 d = -0.95（D 太强）：reward = 0.05，改进 0.1 后 0.14（提升 0.095 最大，但绝对值仍很低）

但注意：在 d = -0.95 处 G1 改进 0.1 极难——这是 95% 准确率 D 的"非人类区"，G1 几乎过不来。

1.4 D 太强时真正的灾难：完全陷入低分区域

不是"奖励接近 0"（虽然小但有），而是 "G1 怎么努力都到不了高分区"：

区间含义PPO 感受
d ∈ [+0.5, +1]D 认为"像人"style_reward > 0.94，PPO 感觉良好但改进空间小
d ∈ [-0.5, +0.5]D "模糊地带"reward 0.5~0.94，梯度最丰富
d ∈ [-1, -0.5]D 认为"不像人"reward 0.05~0.5，奖励低，G1 改进后仍滞留此区

D 太强把 G1 一直困在 [-1, -0.5] 区间，G1 努力也只能爬到 [-0.7, -0.3]，永远拿不到 0.8+ 的高分。

PPO 看到一个永远 0.1~0.3 的奖励，不知道怎么跳出这个区间——这就是"D 太强导致训练失败"的真正含义。

问题 2: 为什么不设计自带梯度的奖励？

2.1 直觉：好像 (d+1)/2 永远有梯度

假设我们改用：

style_reward = (d + 1) / 2   # d ∈ [-1, 1] → reward ∈ [0, 1]

看起来"永远有梯度"：

  • d = -0.99 时 reward = 0.005（虽然小但不为 0）
  • d = -0.5 时 reward = 0.25
  • d = +0.5 时 reward = 0.75

问题在哪？看 D 太强时：

看起来也很糟糕！d = -0.95 时 reward = 0.025——还是几乎为 0！

2.2 根本限制：D 的输出范围是 [-1, +1]

关键洞察：无论用什么函数，只要 D 的输出有界，G1 收到的奖励就受限于 D 的"宽容度"。

如果 D 太强，无论 f(d) 长什么样，f(远离+1) 都会很小：

奖励函数d=+0.9d=0d=-0.9
1 - 0.25(d-1)²0.990.750.01
(d+1)/20.950.500.05
max(0, d+0.5)1.00.50
log(1+exp(d·5))/log(1+exp(5))0.990.500.01

所有奖励函数在 d 远离 +1 时都很小——这不是函数选错了，是 D 强弱的本质问题。

2.3 更深的问题：奖励函数的"标准"从哪来？

如果不用 D 算奖励，我们需要自己定义"什么是好动作"。这才是根本难题：

A 方案 A：关节角距离

奖励 = -||关节角 - 专家关节角||²

问题： 走路时关节角范围很广，"距离" 难以定义"步态风格"。只能惩罚"姿势偏差"，不能学"运动模式"。

B 方案 B：物理量距离

奖励 = -||base 高度变化 - 专家 base 高度变化||²

问题： 选哪些物理量？步态周期、左右对称、关节协调……每加一个都需要手动调权重。

C 方案 C：分类器 D

奖励 = D(状态) 输出经过变换

优势： D 自己学"什么是好"，不需要人手设计

代价： 奖励依赖 D，D 太强就崩

这就是 AMP 的核心权衡：

  手设计奖励
      ↓
   简单但不准（学不到"风格"）
      ↓
  D 自动学奖励
      ↓
   准确但有"训练平衡"问题 ← 我们在这里

2.4 为什么不"既要 D 又要平滑奖励"？

可以试！比如：

style_reward = (d + 1) / 2 + ε · reward_signal_strength(d)
# 后者是某种"奖励置信度",让 G1 知道"我的动作离像人多远"

但这就需要额外设计"置信度"——又回到了"人手定义"的死循环。

或者可以试：

style_reward = max(0, d)   # 把负数全归零

但 d 永远接近 -1 时，reward 全是 0，等于没训练。

问题 1 vs 问题 2 的关系

两问题的本质相同：都是 D 的输出空间有限 + 强 D 把 G1 推到奖励低区。

总结

问题 1 的精要回答

D 太强 → G1 拿到的 d 远离 +1 → 平方奖励函数 1-0.25(d-1)² 在 d 远离 +1 时绝对值接近 0 → G1 长期拿到 0.05~0.5 的低分 → PPO 看到一直很低的奖励，没有上升空间 → 训练崩溃。

重要纠正：局部梯度（即"斜率"）在 d=-1 处是 1.0（最大），不是 0。问题不是"梯度消失"，而是两个层次：

  • 绝对值小：G1 拿到 0.05，无论怎么改进，绝对值仍很小
  • 难逃低分区：G1 长期在 d ∈ [-1, -0.5]，是 D 判定的"明显不像人"区，改进极难

问题 2 的精要回答

不能"既要 D 又要平滑奖励"，因为：

  1. D 的输出有界 [-1, +1]，任何 f(d) 在 d 远离 +1 时都会很小
  2. 不依赖 D 又想定义"像人"——人手做不到
  3. 加额外"置信度"或"距离度量"——又回到人手设计

AMP 选了"靠 D 自然平衡"：grad_pen 让 D 不会冲到 100% 准确率，G 进步 D 也进步，两者在 70-80% 平衡。这是用神经网络学奖励的必然代价，没有银弹。

两个问题的关系

它们是同一个问题的两面：

  • 问题 1：强 D 怎么让奖励崩？
  • 问题 2：既然崩了为什么不换奖励？

答案相同：D 强弱决定了奖励信号的丰富度，强 D 必然导致奖励枯竭；用 D 就要接受 D 太强的代价。
